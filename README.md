# Bubble-LIamERD-sync

Bubble 프로젝트 실험을 위한 최소 크롬 익스텐션 템플릿입니다. 액션 버튼, 팝업 UI, 백그라운드 서비스 워커, 콘텐츠 스크립트를 모두 포함해 바로 기능을 붙일 수 있습니다.

## 폴더 구조

```
├── manifest.json          # 크롬 익스텐션 설정
└── src
    ├── background.js      # 설치 이벤트 & 액션 클릭 처리
    ├── content-script.js  # 페이지에 뱃지 표시 및 메시지 수신
    └── popup
        ├── popup.html     # 팝업 마크업
        ├── popup.js       # 탭으로 메시지 전송 로직
        └── popup.css      # 간단한 다크 테마 스타일
```

## 주요 동작
- 확장을 설치하면 `chrome.runtime.onInstalled`를 통해 콘솔 로그를 남깁니다.
- 툴바 아이콘을 클릭하거나 팝업 버튼을 누르면 현재 탭의 콘텐츠 스크립트로 `ACTION_CLICKED` 메시지를 전달합니다.
- 콘텐츠 스크립트는 페이지 우측 하단에 "Bubble Ready" 배지를 띄우고, 메시지를 받으면 알림 창을 띄워 동작을 확인할 수 있습니다.

## 개발 및 로드 절차
1. 이 폴더를 그대로 유지한 채 크롬 주소창에 `chrome://extensions`를 입력합니다.
2. 우측 상단의 **개발자 모드**를 켭니다.
3. **압축해제된 확장 프로그램을 로드** 버튼을 눌러 이 프로젝트 루트를 선택합니다.
4. 툴바에 "Bubble Quick Actions" 아이콘이 나타나면 팝업 버튼을 눌러 동작을 확인합니다.
5. 코드 수정 시 `chrome://extensions` 페이지에서 **새로고침** 아이콘을 눌러 변경 사항을 반영합니다.

## 다음 단계 아이디어
- Bubble API 또는 LLamERD 연동 로직을 `background.js`에 추가해 네트워크 호출을 관리할 수 있습니다.
- 콘텐츠 스크립트에서 DOM을 탐색해 필요한 UI를 삽입하거나, 팝업에서 폼을 추가해 사용자 입력을 받아보세요.
- `chrome.storage`를 통해 사용자 설정(예: 토글 옵션, API 키 등)을 저장하면 더 풍부한 확장 프로그램을 만들 수 있습니다.
